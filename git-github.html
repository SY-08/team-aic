Git基礎第1話 Gitってなに？＝作業のセーブ機能
——スカイブルーの世界。Gitさんは、自分が何者かを思い出せずにいた。
Gitさん
神様…。おれ「Git」って名前だけど、結局なにをする者なの？
神様
ふむ。ひとことで言えば、お前は「作業のセーブ機能」じゃ。
Gitさん
セーブ…？ゲームの、あのセーブ？
神様
そうじゃ。文章でもコードでも、区切りのいいところで「ここまで！」と記録を残す。それがお前の力じゃ。
GitHub氏
へぇ。じゃあ失敗しても、前のセーブに戻れるってこと？
神様
その通り。だから安心して挑戦できる。消えて困るものを、時間ごと残せるんじゃ。
Gitさん
なるほど…！おれ、けっこう頼れるやつだったんだ。
神様
図で見るとこうじゃ。作業して、節目でセーブ。これを繰り返すだけ。
Git基礎第2話 コミット＝セーブする
翌朝。Gitさんは「セーブのやり方」を知りたくなった。
Gitさん
神様、セーブって具体的にどうやるの？
神様
その1回のセーブを「コミット」と呼ぶ。今の状態を写真に撮って、名前を付けて保存するイメージじゃ。
GitHub氏
名前？
神様
「ここで何をしたか」を一言添える。例えば『トップページの文言を修正』のようにな。
Gitさん
あとで見たとき、何のセーブか分かるように…！
神様
その通り。コミットが積み重なると、作業の歴史そのものになる。
GitHub氏
日記みたいだね。
神様
よい例えじゃ。1コミット＝1ページの日記。図を見てみよ。
GitHub基礎第3話 GitHub＝みんなで見られる保管庫
セーブを覚えたGitさん。でも、GitHub氏はまだ自分が何者か分からない。
GitHub氏
神様…。おれ「GitHub」だけど、Gitと何が違うの？
神様
Gitが手元でのセーブなら、GitHubは「そのセーブを、みんなが見られる場所に置く保管庫」じゃ。
Gitさん
おれのセーブを、ネットに置いておく所…？
神様
そう。自分のパソコンが壊れても大丈夫。仲間と一緒に見て、直せる。
GitHub氏
だからか…！なんかおれの中に、おれじゃない誰かがたくさんいる気がしてたんだよー。
神様
team AICのホームページも、GitHubに置いたものが世界に公開されておる。
Gitさん
手元でセーブ（Git）、ネットで共有（GitHub）。コンビなんだね。
神様
その通り。二人で一人前じゃ。図を見よ。
