# みんなのAI教室：教材生成の運用

「08-15 勉強会活動」は、公開ページ「みんなのAI教室」の正本ページです。
勉強会の本文・企画・教材の基準はNotionに保管し、確認済みの内容だけをGitHub Pagesへ反映します。

## 教材ができるまで

```text
活動共有ノート・08-06 実行ログ
        ↓
GPTタスクが実作業を読み取る
        ↓
対象者を1つ選び、教材を1本作る
        ↓
確認待ちとしてNotionに保存する
        ↓
人が内容・安全性・公開可否を確認する
        ↓
確認済みの内容だけを公開ページへ反映する
```

## 対象者の順番

1. 個人商店
2. ビジネスパーソン
3. 高齢者
4. 小学生

1回の更新につき原則1本を作り、次回は次の対象者へ進みます。活動内容が対象者に合わないときは、無理に教材化せず理由を記録します。

現在の順番は [`generation-state.json`](./generation-state.json) で管理します。

## ファイルの役割

- [`gpt-task-prompt.md`](./gpt-task-prompt.md)：GPTに渡す教材生成プロンプト
- [`material-template.md`](./material-template.md)：1本分の出力形式
- `drafts/`：生成直後の教材。すべて「確認待ち」
- `approved/`：人が確認し、公開可能と判断した教材
- `fixtures/`：テスト用に一般化した活動記録
- [`generation-state.json`](./generation-state.json)：直前・次回の対象者と履歴

教材本文の正本はNotionです。GitHub側のファイルは、生成形式、確認処理、公開反映のための実装とレビュー用の控えです。

## 事実と一般化を分ける

教材では、次の2つを明示的に分けます。

- **実際に行った作業**：活動共有ノートや実行ログに記録された内容
- **生活・仕事への置き換え**：参加者が試すための一般化した例

記録にないことを、team AICの実績として追加しません。

## 公開前レビュー

生成直後は必ず `公開状況：確認待ち` です。次を人が確認してから公開対象にします。

- 元の活動記録と一致しているか
- 推測を実績として書いていないか
- 個人情報、秘密情報、認証情報がないか
- 対象者に合った難易度か
- デモが安全に試せるか
- 健康、法律、金銭、契約などをAIだけで判断させていないか

`scripts/seminar-draft.mjs` は、必須見出し、対象者の順番、公開状況、秘密情報らしい文字列を機械的に確認します。

## テスト

```powershell
node scripts/seminar-draft.mjs --check-all
```

このチェックは教材を公開状態に変更しません。
