Open Source / Automation

Automation

Notion「08 Website CMS」を正本として、GitHub・GitHub Pagesへ自動的に反映する仕組みをまとめています。

全体の流れ

Notionが頭脳、GitHubが中継、GitHub Pagesが公開先という役割分担で動いています。

🧠 Notion

「08 Website CMS」に各ページの内容をまとめ、正本として管理します。

🤖 GitHub Actions

Notion APIで内容を取得し、HTMLへ変換してこのリポジトリに反映します。

🌐 GitHub Pages

変更があれば自動でコミットされ、公開サイトに反映されます。

仕組みの詳細（Notion同期）

08-09配下の各ページから、以下の3セクションが自動同期されます。

08-09-01 Notion to GitHub

Notion to GitHub

GitHub Section

automation.html#notion-to-github

目的

08 Website CMSを正本として、GitHub Pagesに反映する。

流れ

08 Website CMS
↓
GitHub Actions
↓
Notion API
↓
HTML生成
↓
GitHubへcommit / push
↓
GitHub Pages公開

方針

08を編集した内容を、下書き状態でも公開してよい。

タイムリーさを優先する。

08-09-02 GitHub Actions

GitHub Actions

GitHub Section

automation.html#github-actions

目的

人がClaude Codeを毎回起動しなくても、GitHub上で同期処理を自動実行する。

起動方法

GitHub Actions
│
├─ schedule：5分おきに実行
├─ workflow_dispatch：手動実行
└─ repository_dispatch：将来の外部イベント起動

必要なSecrets

NOTION_TOKEN
NOTION_WEBSITE_CMS_ID
08-09-03 note / SNS / YouTube / Book連動

note / SNS / YouTube / Book連動

GitHub Section

automation.html#media-sync

目的

08やGitHub Pagesの内容を、note、Instagram、X、YouTube、本化へつなげる。

流れ

GitHub Pages公開内容
│
├─ note下書き
├─ Instagram投稿文
├─ X投稿文
├─ YouTube台本
└─ Book Materials

方針

完全自動投稿ではなく、まずは下書き・台本・投稿文の自動生成を行う。

投稿は人がタイミングを見て行う。

使用しているGitHub Secrets

🔑
NOTION_TOKEN と NOTION_08_WEBSITE_CMS_PAGE_ID をGitHub Secretsに登録して使用しています。 値そのものはコードにもこのページにも一切記載していません。

安全のための仕組み

🔒
同期スクリプトは、APIキー・トークン・パスワードらしき文字列や、個人情報・守秘義務情報らしき文字列を検知した場合、 そのページの出力を行わず、同期ログ（docs/notion-sync/latest-sync-log.md）に警告として記録します。 詳しい仕組みは docs/automation/notion-to-github-full-sync.md にまとめています。
