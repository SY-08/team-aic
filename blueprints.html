最新追記：自動化の実行時間とSync運用

team AICの自動化は、無料運用を前提にする。

GitHub ActionsのNotion Syncは、5分ごとではなく1時間ごとを基本にする。

Notion Sync
読み方：シンク
意味：同期
推奨頻度：1時間ごと
cron：0 * * * *

無料運用の考え方

軽い処理
│
├─ Notionを読む
├─ Markdown / HTMLに変換
├─ GitHubへcommit / push
└─ GitHub Pagesを更新

これらはGitHub Actionsで行う。

重い処理
│
├─ 画像生成
├─ 動画生成
├─ 大量のAI生成
└─ 長時間の分析

これらはGitHub Actionsではなく、ChatGPT、Claude、GeminiなどのAI側で行う。

エラー対策

今回のpush競合を防ぐため、Notion Syncはpush前にremote mainを取り込む。

git fetch origin main
git rebase origin/main
変更があればcommit / push

同時実行を防ぐため、concurrencyを使う。
