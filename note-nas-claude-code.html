🟢 やさしい版（はじめての人へ）

ぜんぶ「家に置いたゲーム機」でたとえます。まず“たとえの約束”を先に見ると、迷わず読めます。

🎮 たとえの約束（対応表）この記事で統一

今日やったこと（ひとことで）

家に置いた「ゲーム機の本体」（NAS）の中でAIというソフトを動かし、パソコンやスマホのコントローラーからいつでも呼べるようにして、手元のセーブ（作業フォルダ）と自動でそろえられるようにした一日。

今日のゴール（4つ）

  • ゲーム機の本体（NAS）に、AIというソフトを入れて動かす
  • パソコンとスマホ、どちらのコントローラーからもそのソフトを呼べるようにする
  • ケーブルを抜いても、ソフトが本体の中で動き続けるようにする（＝本体に内蔵電源＝tmux）
  • 本体と手元で、セーブ（作業フォルダ）を自動でそろえる

やったこと（順番に）

  • パソコンを“コントローラー”にして、家のゲーム機の本体に遠くからつなぐ。
  • その本体の中に、AIというソフトを入れて動かす。
  • スマホも“コントローラー”として、同じソフトを呼べるように設定。
  • ケーブルを抜いてもソフトが動き続ける“本体の内蔵電源”（tmux）をON。
  • 本体と手元で、セーブ（作業フォルダ）を自動でそろえる係を用意。
  • 実際にマンガ一式（約1.5GB）が、まるっと本体にそろったのを確認。

なぜこの4つが大事か（3つのポイント）

① 本体はゲーム機、AIはソフト

NASは家に置いた“ゲーム機の本体”。ずっと電源を入れておけるから、いつでも働いてもらえる。AI（Claude）は、その本体で動かす“ソフト”。本体には画面をつながないので、操作はパソコンやスマホという“コントローラー”を使って遠くから行う。

② なぜ最初は「ケーブルを抜くと止まる」のか

最初、コントローラー（パソコン）と本体をつなぐケーブル（SSH）は、“操作”だけでなく“電源”も一緒に送っている特殊なケーブル。だからケーブルを抜くと、電源も切れて、ソフト（AI）も止まってしまう。ここが最初の状態。

③ 止まらないようにする＝本体に「内蔵電源」を足す（tmux）

そこで、本体の中に“消えない電源”（tmux）を用意する。電源が本体から出るようになるので、コントローラーのケーブルを抜いても、ソフトは本体の中で動き続ける。あとはパソコンでもスマホでも、好きなときにコントローラーを差して操作すればいい。これで“どこにいてもAIを呼べる”状態になる。

④ 2つの場所でセーブをそろえる（Syncthing）

本体（NAS）と手元（パソコン）で、“セーブデータ”（作業フォルダ）をいつも同じ中身にそろえる係（Syncthing）を入れた。片方でセーブを更新すると、もう片方にも自動で反映される。だから手元で描いても、本体のAIが作っても、両方いつも同じ中身。

今日の気持ち

はじめての言葉ばかりだったけど、ぜんぶ“ゲーム機（本体＋ソフト＋コントローラー）”にたとえたら、驚くほどスッと分かった。むずかしいものは、知っているものに置きかえればいい。しかも、たとえが最後まで筋道どおりだと、途中で迷わず理解が進む——そう実感できた一日。

🔧 くわしい版（用語そのまま）

今日の活動テーマ

  • 自宅のUGREEN NAS（DXPシリーズ）に、Claude Code（ターミナルで動くAIの相棒）を導入する
  • スマホのClaudeアプリから、NAS上のClaudeを操作できるようにする
  • PCを閉じてもNAS上のClaudeが動き続ける「常時起動」を用意する
  • ローカルPCとNASで、マンガ制作フォルダ（manga-production）を自動同期する土台をつくる

今日やったこと（コマンド再現）

実際の端末画面ではなく、打ったコマンドを再現ブロックで載せます（生のスクリーンショットは使いません）。

1. NASにSSH接続

WindowsのPowerShellから、モニターの無いNASにログインして遠隔操作します。住所は家の中だけで通じる私設IP（192.168.x.x の形）。

▷ PowerShell（PC）
PS> ssh 管理者アカウント@192.168.x.x # NAS（Linux）のシェルに入る

2. Claude Code の導入

NAS上でインストールし、置き場所 ~/.local/bin をPATHに登録。claude の一言で起動できるように。

▷ NAS（Linux シェル）
$ curl -fsSL https://claude.ai/install.sh | bash $ export PATH="$HOME/.local/bin:$PATH" $ claude # AIの相棒を起動

3. スマホから操作（Remote Control）

Claude内で /remote-control を有効化。スマホのClaudeアプリから同じセッションに接続でき、処理はNASで動き続けます（スマホは“リモコン”）。

▷ Claude 内
> /remote-control # 表示された案内からスマホのClaudeアプリで接続（接続URLは非公開）

4. 常時起動（tmux）

SSHは「操作＋電源の2in1ケーブル」。抜くとClaudeも止まります。tmux（＝NASの中に据えた“消えない電源コンセント”）の中でClaudeを動かすと、電源（tmux）と操作（PC・スマホ）が分離され、ケーブルを抜いてもNAS上で動き続けます。

▷ NAS（Linux シェル）
$ sudo apt install tmux $ tmux # 消えない電源の中へ $ claude # この中で起動 → 切っても動き続ける # 離脱: Ctrl+b → d ／ 復帰: tmux attach

5. Docker＋Syncthing（NAS側）

UGOSのDockerで linuxserver/syncthing のコンテナを作成。GUI（管理画面）で設定しました。

🐳 Docker コンテナ設定UGOS 管理画面（GUI）再現
イメージlinuxserver/syncthing
権限（PUID/PGID）自分のユーザーIDに合わせる
ポート8384（設定画面） / 22000（同期通信）
フォルダ/data（同期対象を割り当て）

6. Syncthing（PC側）とペアリング

▷ PowerShell（PC）
PS> winget install GermanCoding.SyncTrayzor
🔗 デバイス ペアリング＆同期Syncthing 画面（GUI）再現
ペア方法お互いのデバイスIDを登録（IDは非公開）
同期フォルダmanga-production（約1.5GB／3,245ファイル）
結果PC→NAS へ完全同期。両方「最新」を確認
おまけ：ソフトを入れる“調達係（パッケージマネージャー）”は、Linuxは apt、Windowsは winget。OSが違うだけで役割は同じ（sudo apt install 〇〇 ≒ winget install 〇〇）。

学び（やさしい説明）

NASとは

NAS（Network Attached Storage）は「家に置く、電源をつけっぱなしにできる小さなサーバー」。据え置きゲーム機の本体のようなもので、モニターやキーボードを普段つながないため、操作は別のPCから遠隔で行う。SSH（Secure Shell）は「離れたコンピュータのコマンド画面を、暗号化された安全な回線で操作する方法」。住所は家の中だけで通じる私設IP（192.168.x.x の形）。

CLIとGUIとは

  • CLI（Command Line Interface）：黒い画面に文字でコマンドを打つ方式。細かい作業・自動化に強い。今日のNAS操作やインストールはこれ。
  • GUI（Graphical User Interface）：アイコンやボタンをマウスで操作する方式。直感的。UGOSの管理画面やSyncthingの設定画面はこれ。
  • 同じNASを「文字で命令する（CLI）」か「画面を触る（GUI）」かの違い。今日は両方を行き来した。

SyncthingとDocker

Syncthingは「2台の機器のフォルダを、つねに同じ中身に保ち続ける自動同期ツール」。クラウドを介さず、機器同士が直接・暗号化してやり取りする（P2P）。片方が壊れても、もう片方に全データが残る。NASは常時稼働、PCは起動したときに差分を自動で追いつく。Dockerは「本体を汚さず、隔離した小部屋（コンテナ）でアプリを動かす仕組み」。イメージ（設計図＝カセット）からコンテナ（動く実体）を起動し、ポート（番号付き端子）で設定画面(8384)やデータ(22000)につなぐ。コンテナのPUID/PGIDを自分の番号にそろえ、同期ファイルをNAS上のClaudeも読み書きできるようにした。

活動を経てのジャーナリング

今日はNASという「自分の基地」を、遠隔から一日かけて立ち上げた。いちばん腑に落ちたのは、『操作』と『電源』を分けるという発想だった。SSHという1本のケーブルに操作と電源が同居していると、線を抜いた瞬間にすべて止まる。でもtmuxで電源を内蔵にしてしまえば、操作する側（PC・スマホ）は好きに抜き挿しできる。これは道具の話にとどまらず、『動かし続けたいものは、目の前の自分の接続に依存させない』という設計の考え方そのものだと感じた。

もうひとつは、比喩で理解することの強さ。ただし、比喩は最後まで筋道が通っていて初めて効く。NASを据え置きゲーム機、SSHを電源つきコントローラーケーブル、tmuxを本体内蔵の電源……と一貫して翻訳したら、初めての概念が驚くほどすっと入った。分からないものは、知っているものに置き換えればいい。

最後に、今日作ったのは『マンガを描く仕組み』そのものではなく『どこにいても創作を進められる土台』だということ。家のNASが常に起き、スマホから指示でき、PCと自動で同期する——場所と時間の制約が一枚はがれた感覚がある。次は、この土台の上で実際に4話を動かしてみたい。

次にやること

  • PC側Syncthingに任意でGUIパスワードを設定する。
