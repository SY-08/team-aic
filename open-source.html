Notion更新で育つ設計図

設計図や考え方は、Notionを更新するたびに公開ページへ反映される前提で運用します。

Publishing Flow

正本はNotion。公開はGitHub Pages。GitHubは中継。

横塚さんがGPT / Claudeに相談する
↓
公開できる内容をNotion「08 Website CMS」に整理する
↓
GitHub ActionsがNotionの内容を同期する
↓
GitHubに変更履歴として残る
↓
GitHub Pagesの公式サイトに反映される

この流れにすると、相談、実装、振り返り、公開がバラバラにならず、後から追える活動記録として残ります。
