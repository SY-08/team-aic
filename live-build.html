決めたこと

勉強会は、流行りのAI紹介ではなく、team AICが実際にやった作業から派生させます。

  • → 活動共有ノートを、振り返りと教材化の元データにします
  • → 最初は文章ベースで公開し、あとからスライドや配布資料を追加します
  • → 参加者がその場でできる小さなデモを必ず入れます

次にやること

2026-07-07

ホームページとNotionの自動連携のしくみを作った

今日やったこと

ホームページの原稿をNotionに書くだけで、自動でサイトに反映される仕組みの「土台」を作りました。

  • → ホームページの大もとの原稿を、Notionの1か所にまとめる方針にしました（Notion「08 Website CMS」を正本とする同期のしくみを設計）
  • → Notionに書いた文章を、自動でホームページの形に変換するプログラムを作りました（ページ本文をHTMLへ変換する scripts/notion-sync.mjs）
  • → 決まった時間に自動で反映する“自動処理”を用意しました。手動でも動かせます（GitHub Actions .github/workflows/notion-sync.yml）
  • → ホームページのメニューの並びを、Notionの構成に合わせて整理しました（08の階層構造をナビゲーションに反映）
  • → 各ページに「Notionから自動で反映しています」という表示を付けました（Synced from Notion バッジを追加）
  • → パスワードなど公開してはいけない情報を見つけたら、自動で止める安全装置を付けました（公開NGパターン検知で出力をスキップ）

決めたこと

これからはホームページを直接いじらず、Notionを編集して自動で反映する運用にします。

  • → ホームページの更新は、Notionを編集して反映する形にします（HPをGitHubで直接編集せず、Notion「08 Website CMS」を正本に同期）
  • → 変わったところがある時だけ記録します。無駄な更新は作りません（差分がある場合のみコミット）
  • → 出してはいけない言葉が入ったページは公開せず、記録に注意書きを残します（公開NG検知でスキップし同期ログに警告）

次にやること

このあと、Notionと自動反映をつなぐ設定をして、実際に反映されるか確かめます。

  • → NotionとGitHubをつなぐ“合いカギ”を登録します（GitHub Secrets に NOTION_TOKEN / NOTION_08_WEBSITE_CMS_PAGE_ID を設定）
  • → 自動処理を1回手で動かして、ちゃんと反映されるか確かめます（Actionsを手動実行し初回同期を確認）
  • → Notion側の各ページを整えて、反映結果を確認します（08配下の各ページを整備）

team AIC 公式サイトを公開した

今日やったこと

team AICの公式ホームページを作って、インターネット上に公開しました。

  • → ホームページの土台一式を作りました（HTML / CSS / JS の初期サイト）
  • → 白ベースのやさしいデザインにしました
  • → トップ・team AICについて・活動内容・プロフィールなど計13ページを作りました
  • → 記録・下書き・設計図をためる置き場を用意しました（docs 配下）
  • → 公開してはいけない情報のチェックリストを作りました

決めたこと

「完成してから出す」のではなく、「作りながら公開する」進め方にしました。

  • → デザインは、白ベースのやさしい印象にします
  • → 完成記事ではなく、作っている途中の記録を公開します
  • → Notionで考え、GitHubで中継し、ホームページで公開する、という役割分担にします

次にやること

Notionとホームページのつなぎ方を整理して、少しずつ中身を増やしていきます。

  • → Notionからホームページへ反映する流れを整理して確認します
  • → 設計図・プロンプト・本化素材・動画下書きの中身を少しずつためます
  • → SNSでシェアした時に出る画像やファビコンなどの素材を用意します（OGP画像・ファビコン）
  • → noteへ初めて転記するタイミングを検討します
