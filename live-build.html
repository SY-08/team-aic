2026-07-07

ホームページとNotionの自動連携のしくみを作った

今日やったこと

ホームページの原稿をNotionに書くだけで、自動でサイトに反映される仕組みの「土台」を作りました。

  • → ホームページのおおもとの原稿をNotion「08 Website CMS」に置く方針を決め、同期のしくみを設計
  • → Notionの本文を自動でホームページの形（HTML）に変換するプログラム scripts/notion-sync.mjs を作成
  • → 決まった時間に自動反映するGitHubの自動処理 .github/workflows/notion-sync.yml を作成（手動実行も可能）
  • → ページのメニュー構成をNotionの並びに合わせて整理
  • → 各ページに「Notionから自動反映中」の表示を追加
  • → 公開してはいけない情報（パスワード等）を見つけたら自動で止める安全装置を実装

決めたこと

これからはホームページを直接いじらず、Notionを編集して自動で反映する運用にします。

  • → 今後、HPの更新はGitHubを直接編集せず、Notion「08 Website CMS」を編集して同期する運用に切り替える
  • → 同期は変更があるときだけ記録する（無駄な更新を作らない）
  • → 公開NGらしい文字列を見つけたページは出さず、記録に警告として残す

次にやること

このあと、Notionと自動反映をつなぐ設定をして、実際に反映されるか確かめます。

  • → GitHubの秘密設定（NOTION_TOKEN / NOTION_08_WEBSITE_CMS_PAGE_ID）を登録する
  • → 自動処理を手動実行して、初回の同期が成功するか確認する
  • → Notionの08配下の各ページを実際に整えて、反映結果を確認する
2026-07-07

TEAM AIC 公式サイトを公開した

今日やったこと

TEAM AICの公式ホームページを作って、インターネット上に公開しました。

  • → ホームページの初期サイト（HTML / CSS / JS）を作成
  • → 白ベースのやさしいデザインを実装
  • → トップ・私たちについて・活動内容・メンバーなど計13ページを作成
  • → 記録・下書き・設計図をためる置き場を用意
  • → 公開してはいけない情報のチェックリストを作成

決めたこと

「完成してから出す」のではなく、「作りながら公開する」進め方にしました。

  • → デザイン方針：白ベースのやさしい印象
  • → 公開方針：完成記事ではなく、作っている途中の記録を公開する
  • → 構成方針：Notionで考え、GitHubで中継し、ホームページで公開する役割分担

次にやること

Notionとホームページのつなぎ方を整理して、少しずつ中身を増やしていきます。

  • → Notion側の運用ルールを整理し、ホームページへの反映の流れを確認する
  • → 設計図・プロンプト・本化素材・動画下書きの中身を少しずつためる
  • → OGP画像・ファビコンなどの素材を用意する
  • → noteへの初回転記のタイミングを検討する
